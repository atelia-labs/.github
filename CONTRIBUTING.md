# Contributing to Atelia Labs

Atelia is early and public. Contributions are welcome through issues,
discussions, design notes, and pull requests.

## Where To Start

- Use the relevant repository for implementation issues and pull requests.
- Use [`atelia`](https://github.com/atelia-labs/atelia) for project-wide
  philosophy, specifications, and governance.
- Use discussions in `atelia` for open-ended questions and proposals.

## Workflow

GitHub is the public surface for Atelia Labs. It holds code, pull requests,
issues, discussions, releases, security notices, CI results, and public roadmap
signals.

Linear is the execution surface. Maintainers and AI agents use it for work
decomposition, prioritization, assignment, handoff, implementation tracking, and
agent work queues.

Public input should start in GitHub. Accepted work may be tracked in Linear when
it needs execution management. Public decisions should be reflected back into
GitHub issues, discussions, docs, pull requests, or release notes.

## Pull Requests

Keep pull requests focused. Include:

- the problem being solved;
- the changed files or docs;
- any test or review performed;
- follow-up work, if any.
